Bluecherry version 2 beta released

After several rounds of delays we are proud to announce the beta public of version 2.  This release has been in the works for well over a year and incorporates more then 100 features and improvements from version 1.  Version 2 is completely written and absolutely none of the version 1 code reused.  Version 2 focuses on performance, security enhancements, and using existing Linux kernel APIs.  This new code base will allow us to fix bugs faster and add features much more quickly.

If you do not have an existing Bluecherry series MPEG-4 compression card, then you can order one from the link below.  If you purchase a card and you decide to return it after the beta period you will not be charged a restocking fee (assuming everything is returned).  Beta testers who submit *regular* feedback (bug reports, feature requests, suggestions,constructive criticism) will receive discounts on version 2 licenses or store credit towards Bluecherry products.

We have created a 15% off coupon code for the products listed below.  You must use the coupon code BluecherryV2Beta-7YhCkA at the time of the order to receive this discount

http://store.bluecherry.net/categories/Capture-Cards/Video4Linux-supported/Hardware-compression-(MPEG4)/

I would like to take this opportunity to thank our lead developers, Ben Collins, John Brooks, Alex Belykh and Ben Winslow for their hard work and input as version 2 developed.  Without their dedication this beta version would never have not been released.

Below are details on how to download the beta.  *Please* do not call or open support tickets.  Instead post any questions, comments and suggestions on the web forums (http://forums.bluecherrydvr.com/index.php?/forum/14-general-discussion) or via IRC (irc.freenode.com/bluecherry or http://webchat.freenode.net?channels=bluecherry&uio=MTI9dHJ1ZQ64).  If you locate a bug please post the bug report on the forums or IRC with way(s) to replicate the issue.  We will review it and either add the bug for you, or ask that you add the bug on improve.bluecherrydvr.com.  If the client crashes a report will automatically be sent to our server, but it would be nice to go ahead and report the issue on the forums or IRC.

Disclaimer: This is a *beta* release.  Also keep in mind that database schema changes may render recorded events inaccessible.  In a nut shell, we aren’t liable if something bad happens, if you lose events or events aren’t recorded properly.

This release is still lacking several features that may be important to you.  Below is the list of features that made the cutting room floor in order to get this release out today, instead of delaying it (again).  The majority of the items listed below are posted on the ‘during beta’ section of the version 2 roadmap (http://improve.bluecherrydvr.com/versions/show/43) and will be added as we go along the beta.

Major features in this release:

  • Stacking of multiple cards
  • Linux client (Ubuntu 10.04)
  • Windows client (Support for XP, Vista and 7)
  • OS X client (10.5 / 10.6)
  • Client is no longer fixed to 1280×1024 resolution
  • Drag and drop for both the client and web interface
  • Headless server (The server no longer requires an X session to run)
  • Significantly increased performance on the server
  • Improved motion mapping
  • Only one port to forward (Everything operates on TCP port 7001)
  • Network camera support
  • H.264 hardware compression card support

A full list of features and bugs can be found here: (http://improve.bluecherrydvr.com/versions/show/47)

Download instructions:

Download a copy of Ubuntu 10.04: http://www.ubuntu.com/desktop/get-ubuntu/download (Note: The latest is 10.10, make sure to select Ubuntu 10.04 LTS – Long term stable)

Once installed, run this command:

sudo wget --output-document=/etc/apt/sources.list.d/bluecherry.list http://ubuntu.bluecherrydvr.com/sources.list.d/bluecherry.list
sudo apt-get update ; sudo apt-get install bluecherry solo6010-dkms

Start the client and put in the IP address of the v2 installation.  The default login and password is Admin / bluecherry
To manage the settings login with a web browser to: https://ipaddress:7001 The default login and password is Admin / bluecherry

You will receive a non-fatal warning about a untrusted SSL certificate.  It’s safe to ignore this.

Optional, if you want the Linux client installed: sudo apt-get install bluecherry-client

Client download locations:

www.bluecherrydvr.com/downloads

Configuration

Most setting changes saved in the web interface are applied automatically so you do not have to restart the server after each change.  The setup interface should be laid out well enough to get an idea on how to add schedules and motion mapping.  If you require assistance post a question on the forums.
